產品概要

Diamond Track 是給棒球／壘球隊使用的 Web 應用：教練可管理 roster、發布指示、點名、登錄測驗與比賽數據；球員可填寫訓練紀錄、查看指示與個人成績。全系統以 Google 帳號登入，資料儲存在雲端，電腦與手機瀏覽器皆可使用。

角色一覽

教練 COACH建立或管理球隊、審核球員、完整後台功能。

球員 PLAYER加入球隊後使用訓練與個人功能；新加入時可能需等待教練核准。

登入與首次設定

Google 登入

1開啟系統網址後，點選「登入球隊」。

2依畫面完成 Google 帳號選擇與授權。

內建瀏覽器（LINE、Facebook、IG 等）
Google 登入可能被阻擋。請改用右上角選單的「在瀏覽器中開啟」，或使用 Chrome／Safari／Edge 直接開啟網址。

首次進入：選擇身分與球隊

第一次登入且尚無個人檔案時，會進入歡迎／註冊流程：

  1. 選擇身分：教練或球員。
  2. 球隊：教練可建立新球隊並命名；球員則從清單選擇既有球隊加入。
  3. 確認後系統會建立您的使用者資料。球員狀態為「待審核」時，需等待教練核准後才能使用主功能。
若網域未加入 Firebase 授權名單，登入後可能出現 auth/unauthorized-domain。請由專案管理員在 Firebase 後台「Authentication → 設定 → 授權網域」加入正式網址（例如 *.pages.dev 或自訂網域）。

教練：儀表板與設定

COACH

教練主頁（儀表板）

集中顯示：

  • 訊息通知：未讀提示，可標示已讀並跳轉至指示。
  • 球員總數／本月練球出席人數：快速掌握參與度。
  • 團隊戰力分析：雷達圖與本月／近三月平均對照。
  • 全隊最近訓練：週統計長條圖，可進入訓練日誌。
  • 最近訓練紀錄摘要、已發布指示與完成度。

球隊與個人設定

於側欄個人區塊點選設定圖示（教練）：可修改個人顯示名稱、球隊名稱、球隊 Logo（支援圖片網址或上傳檔案）。儲存後全隊介面會同步顯示更新後的隊名與 Logo。

儀表板為讀取較密集的頁面；若同隊多人同時在線，建議教練僅在需要時停留，並避免在極短時間內重複整理頁面，以維持良好體驗。

教練：球員總覽與管理

COACH

球員總覽

檢視全隊名單、基本資料與近期訓練／出席關聯資訊。可由此進入與單一球員相關的操作或檢視（依畫面設計）。

球員管理

處理待審核的加入申請：核准後球員即可使用完整功能。亦可在此類畫面維護球員欄位（依實際表單為準）。

給教練的實務建議

  • 新球季開始前先請全隊完成登入與入隊，減少賽中補資料壓力。
  • 固定一位主要管理員維護 roster 與評分標準，資料較一致。
  • 球員若更換 Google 帳號，需以新帳號重新入隊並由教練重新對應名單。

教練：訓練日誌與指示

COACH

訓練日誌

以日期為軸檢視全隊訓練紀錄，方便回顧課表執行狀況。可與儀表板「週統計」或「最近紀錄」互相跳轉對照。

教練指示

建立給全隊、指定球員或群組的指示內容；可追蹤閱讀與完成回報。與訓練紀錄連動時，可依指示檢視球員回饋與執行情形。

  • 發布後球員側會收到對應通知（依系統邏輯）。
  • 完成度以指示對象人數與實際完成紀錄計算。

訊息與通知習慣

建議指示標題簡短、內文明確截止或場次；重大公告可固定使用「全隊」對象，避免遺漏。

教練：出席與數據登錄

COACH

出席點名

依練習日建立或更新出席紀錄，標記出席與缺席成員。資料可供儀表板「本月練球出席」等統計參考。

數據登錄（比賽與測驗）

登錄比賽或測驗類成績欄位（打擊、投球、守備等，依球隊設定）。送出後會與球員個人檔案／團隊分析連動。

評分標準

於「評分標準」維護隊上共用的分數或等第尺規，讓教練與球員對「好壞」有一致解讀，減少爭議。

大量數字建議在練習或賽後當日完成輸入，避免累積忘漏；若需修正，請依畫面上的編輯流程更新並確認已儲存。

球員專區功能

PLAYER

訓練記錄

球員可登打個人訓練內容、類型與說明；教練可於後台檢視並給予回饋（依畫面欄位為準）。

教練指示

查看指派給自己或全隊的指示，並依要求完成與回報。完成後教練端可看到完成度變化。

比賽與測驗成績、個人檔案

檢視已登錄之比賽／測驗結果與歷史趨勢；於個人檔案維護可編輯之資料（如簡介、守位偏好等，依表單為準）。

待審核狀態

新加入球隊後若顯示「等待核准」，請聯絡教練審核。期間可登出，核准後重新登入即可進入主系統。

常見問題與附錄

常見問題（FAQ）

Q：登入沒反應或沒跳出 Google？
A：改用系統瀏覽器、關閉阻擋彈出式視窗，或於 LINE／FB 內改用「在瀏覽器中開啟」。

Q：顯示 unauthorized-domain？
A：為網域未授權。請管理員於 Firebase 授權網域加入目前網站網址。

Q：資料讀取上限（Quota）？
A：免費方案可能有讀寫上限；畫面若提示已達上限，請稍後再試或請管理員升級方案／優化查詢。

Q：手機版面與電腦不同？
A：本系統為響應式設計，功能相同，選單改為右上角展開。

附錄：技術環境（簡述）

建議使用最新版 Chrome、Edge 或 Safari。需穩定網路。部署於 Cloudflare Pages 時，正式網址應加入 Firebase 授權網域。
